## 4. Residual debts

### 4.1 Macro brief coverage of GB is weak

The v3 macro brief mentions US (+4.7%) and China (+4.7%) import flows but does not carry a dedicated line for GB exports. §1.19 requires symmetry; the LLM chose to use the partner context for imports rather than exports. Mitigations to consider: strengthen the macro_brief.md prompt with an explicit example showing three partner lines, or add a deterministic quality check that scans for `"US"`, `"CN"`, `"GB"`. Not blocking for v3 shipment, but v4 should tighten.

### 4.2 Dashboard hardcoded narrative and missing yoy_country charts

The 3-view dashboard template carries hardcoded panel text (key takeaway, sector context). These read as correct against the fiche data but do not track the regenerated macro brief prose. Two follow-up options: (a) wire the rendered MDX body into each view via slot-by-H2 splitting, or (b) surface a subset of the macro brief frontmatter into the template. Similarly, `output_yoy_country.svg` and `prices_yoy_country.svg` are generated and in the MDX but absent from the dashboard layout.

### 4.3 explain_50pct_variation edge case

For US imports, the computed `cumulative_share = 4.39` produces prose reading "polypeptide hormones explain 439% of the net variation". The cause is offsetting moves across CN8 codes shrinking the net variation below the gross variation. `analysis/enrich_fiches.py` uses `net_variation` when >EUR 0.05 bn, else falls back to `gross_variation`. The threshold did not trigger in this case. Tighten by always capping cumulative_share at 100% or by sanity-checking the ratio before emission.

### 4.4 CN8 label enrichment

`analysis/enrich_fiches.py` currently emits labels as `"{CN8_code} (CN8)"` because the Comext parquet snapshot does not carry descriptive labels. The LLM enriches them from prior context during drafting but this is unreliable. Follow-up: join against the Comext nomenclature table or maintain a local lookup.

### 4.5 chart palette not yet aligned to Cefic April 2021 guidelines

`charts/style.py` still uses `"primary": "#d36b1d"` (legacy). Cefic canonical orange is `#F47B20` (see `reference_cefic_visual_identity.md` memory). All existing charts inherit the legacy palette, so updating this file would require regenerating all charts. Defer to a dedicated styling pass.
