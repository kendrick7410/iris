## 6. Dettes résiduelles

1. **Round-% partiellement obéi.** Le LLM continue d'écrire *"12.0%"*, *"3.0%"*, *"8.0%"*, *"2.0%"*, *"20.0%"* dans v2 (5 violations vs 9 en v1). Fix propre : arrondir upstream dans `indicators.py`/`comext.py` pour émettre un int quand le `.0` est trivial (e.g. `12.0 → 12`). Effort : ~1h, avec tests. Alternative : post-processor regex `(\d+)\.0%` → `\1%` (rapide mais risque de mal toucher des références à des indices type "2.0" non-pourcentage).

2. **Hallucination "Saudi Arabia"** dans trade_imports v2 ligne 79. Le LLM a ajouté un partenaire absent des données. Probable confusion avec les données Singapore ou Corée. À traiter par un contrôle post-draft : extraire les partner labels du fiche et vérifier que chaque partner cité dans le texte est bien dans la liste. Effort : ~2h.

3. **Maladresse chiffre macro brief** (*"€25.1 bn of the €25.1 bn"*). Pas un pattern systématique, mais révélateur d'un risque : quand un chiffre agrégé et un chiffre partiel se ressemblent, le LLM peut les mixer. Mitigation : injecter un fact-check numérique dans le post-draft (vérifier que `sum(parts) ≈ total` quand les deux sont mentionnés). Effort : ~1-2h.

4. **§1.16 drill-down window** : la règle est dans system.md mais aucun test ne vérifie qu'elle est respectée par le pipeline. Si un futur cache Comext produit un drill-down 5-ans dans une section YTD, rien ne l'attrapera en amont. Ajouter une assertion dans `build_trade_fiche()` : si `drill_down.window` ≠ section window, logger un warning et ne pas émettre le drill-down. Effort : ~30min.

5. **Pas de test d'intégration pour §5.3 enforcement.** La fonction `_enforce_fragmented_europe_cap()` est couverte par l'execution du pipeline mais pas par un unit test. Ajouter `tests/test_fragmented_europe_enforcement.py` avec un fixture multi-sections, vérifier la 1ère gardée, les suivantes strippées. Effort : ~30min.

6. **Macro brief v2 fallback "degraded"** disparu — v2 est maintenant quality=ok à 126 mots grâce au word budget 80-150 pour anomaly_active. RAS.
